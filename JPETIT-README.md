    <article class="markdown-body entry-content container-lg" itemprop="text"><h1 dir="auto"><a id="user-content-el-doble-intèrpret-de-jsbach" class="anchor" aria-hidden="true" href="#el-doble-intèrpret-de-jsbach"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>El doble intèrpret de JSBach</h1>
<p dir="auto">Aquesta pàgina descriu la pràctica de GEI-LP (edició 2021-2022 Q2). La vostra tasca és implementar un doble intèrpret per a un llenguatge de programació musical anomenat JSBach. La sortida d'aquest doble intèrpret serà una partitura i uns fitxers de so que reproduiràn la melodia descrita pel compositor.</p>
<p dir="auto">En diem un <em>doble</em> intèrpret perquè funciona en el sentit informàtic (interpreta un programa) i en el sentit musical (interpreta una peça de música).</p>
<p dir="auto"><a target="_blank" rel="noopener noreferrer" href="/jordi-petit/lp-jsbach-2022/blob/main/bach.png"><img src="/jordi-petit/lp-jsbach-2022/raw/main/bach.png" alt="JSBach" style="max-width: 100%;"></a></p>
<h2 dir="auto"><a id="user-content-bach" class="anchor" aria-hidden="true" href="#bach"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Bach</h2>
<p dir="auto">Johann Sebastian Bach (1685-1750) fou un organista i compositor de música barroca. La seva fecunda obra es considera el cim de la música barroca, i una de les màximes expressions de la música universal, no tan sols per la seva profunditat intel·lectual, la seva perfecció tècnica i la seva bellesa artística, sinó també per la síntesi dels diversos estils de la seva època, del passat i per la seva incomparable extensió. És evident que, si hagués nascut a la nostra època, Bach programaria.</p>
<h2 dir="auto"><a id="user-content-presentació-del-llenguatge-jsbach" class="anchor" aria-hidden="true" href="#presentació-del-llenguatge-jsbach"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Presentació del llenguatge JSBach</h2>
<p dir="auto">JSBach és un llenguatge de programació orientat a la composició algorísmica. Amb JSBach s'utilitzen construccions imperatives per generar composicions que donen lloc a partitures que poden ser desades en diferents formats digitals.</p>
<p dir="auto">JSBach té moltes instruccions habituals, però utilitza una sintàxi que és, evidentment, barroca. Aquest és el <em>Hallo Bach</em>:</p>
<div class="snippet-clipboard-content position-relative overflow-auto" data-snippet-clipboard-copy-content="~~~ Kleines Program in JSBach ~~~

Main |:
    &lt;!&gt; &quot;Hallo Bach&quot;
    &lt;:&gt; {B A C}
:|"><pre class="notranslate"><code class="notranslate">~~~ Kleines Program in JSBach ~~~

Main |:
    &lt;!&gt; "Hallo Bach"
    &lt;:&gt; {B A C}
:|
</code></pre></div>
<p dir="auto">Com podeu veure, els comentaris es troben entre triples titlles (<code>~~~</code>). No és necessari escriure els comentaris en alemany, però en Johann ho faria així.</p>
<p dir="auto">Els programes es troben constituïts per procediments (l'ordre no importa) i, per defecte, comencen pel procediment <code>Main</code>.
Cada procediment té un nom, paràmetres (en aquest exemple no n'hi ha), i un bloc de codi associat. Els blocs es troben inscrits
entre els símbols <code>|:</code> i <code>:|</code>.</p>
<p dir="auto">En JSBach, els procediments han de començar per una lletra majúscula. Les variables, en canvi, comencen amb una lletre minúscula. Els noms de notes, també són en majúscules.</p>
<p dir="auto">La primera instrucció del programa <code>&lt;!&gt; "Hallo Bach"</code> és una instrucció d'escriptura (<em>write</em>).  La instrucció d'escriptura no és gaire útil per compondre, però és útil per debugar, perquè permet escriure textos (entre dobles cometes), enters i llistes.</p>
<p dir="auto">La segona instrucció del programa <code>&lt;:&gt; {B A C}</code> és una instrucció de reproducció (<em>play</em>). Aquesta instrucció afegeix la nota o la llista de notes donades a la partitura. Les llistes es donen entre claus amb els seus elements separats per espais. En aquest cas, els elements són les notes músicals <code>B</code>, <code>A</code> i <code>C</code>. JSBach utilitza el sistema de notació musical anglès,
no el sistema de notació musical llatí ni el germànic. Així, aquest programa
genera la melodia Si, La, Do.</p>
<p dir="auto">L'execució del programa anterior produïria la sortida del missatge <code>Hallo Bach</code> per pantalla.
A més, generaria la partitura següent:</p>

<p dir="auto">JSBach permet escriure programes senzills utilitzant enters de forma semblant als LPs habituals. Per exemple, el programa següent mostra com llegir dos nombres i calcular el seu màxim comú divisor utilitzant l'algorisme d'Euclides amb dos procediments i entrada/sortida:</p>

<p dir="auto">Les variables són locals a cada invocació de cada procediment i els procediments es poden comunicar a través de paràmetres. Els procediments llisten els noms dels seus paràmetres formals, però no inclouen els seus tipus. Els paràmetres es separen amb blancs, <del>com Déu mana</del> com en Haskell.</p>
<p dir="auto">Les variables no han de ser declarades, i poden ser de tipus enter o llistes. Les notes músicals, es veurà més endavant, no són altra cosa que constants per a enters.</p>
<p dir="auto">Com es veu a l'exemple, la sintaxi per llegir i escriure és utilitzant <code>&lt;?&gt;</code> i <code>&lt;!&gt;</code> respectivament. L'operador de comparació per igualtat és <code>=</code> i per diferència és <code>/=</code>.  L'assignació es fa amb la instrucció <code>&lt;-</code>.</p>
<p dir="auto">Com no podia ser d'altra manera, el llenguatge de programació JSBach compta amb recursivitat. Aquest programa mostra com solucionar el problema de les Torres de Hanoi:</p>

<p dir="auto">Canviant o afegint més notes a la llista orígen es poden compondre noves peces, ben agradables de sentir!</p>
<p dir="auto">Al programa anterior es poden veure més operacions per a llistes:</p>
<ul dir="auto">
<li>
<p dir="auto"><code>l[i]</code> consulta l'<code>i</code>-èsim element d'una llista <code>l</code>.
Com que JSBach és per a músics, els índexs de les llistes comencen per 1.</p>
</li>
<li>
<p dir="auto"><code>l &lt;&lt; x</code> afegeix l'element <code>x</code> al final de la llista <code>l</code>.</p>
</li>
<li>
<p dir="auto"><code>#l</code> retorna la llargada de la llista <code>l</code>.</p>
</li>
<li>
<p dir="auto"><code>8&lt; l[i]</code> retalla (elimina) l'<code>i</code>-èsim element d'una llista <code>l</code>.
<code>8&lt;</code> es coneix com <em>der Scherenoperator</em> (l'operador de tisores).</p>
<p dir="auto"><strong>Aclariment històric:</strong> És fals que l'operador de tisores fos introduït per Anna Magdalena quan en Johann li volia fer un 13è fill.</p>
</li>
</ul>
<p dir="auto">En JSBach els paràmetres funcionen com en Python (diguem): els enters es passen per còpia, les llistes es passen per referència.</p>
<h1 dir="auto"><a id="user-content-especificació-de-jsbach" class="anchor" aria-hidden="true" href="#especificació-de-jsbach"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Especificació de JSBach</h1>
<p dir="auto">Les instruccions de JSBach són:</p>
<ul dir="auto">
<li>l'assignació amb <code>&lt;-</code>,</li>
<li>la lectura amb <code>&lt;?&gt;</code>,</li>
<li>l'escriptura amb <code>&lt;!&gt;</code>,</li>
<li>la reproducció amb <code>&lt;:&gt;</code>,</li>
<li>la invocació de procediments,</li>
<li>el condicional amb <code>if</code> i potser <code>else</code>,</li>
<li>la iteració amb <code>while</code>,</li>
<li>l'afegit a llistes amb <code>&lt;&lt;</code>.</li>
<li>el tall de llistes amb <code>8&lt;</code>.</li>
</ul>
<p dir="auto">Les instruccions escrites una rera l'altra s'executen seqüencialment.</p>
<h2 dir="auto"><a id="user-content-assignació" class="anchor" aria-hidden="true" href="#assignació"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Assignació</h2>
<p dir="auto">L'assignació ha d'avaluar primer l'expressió a la part dreta del <code>&lt;-</code> i emmagatzemar després el resultat a la variable local a la part esquerra. Exemple: <code>a &lt;- a - b</code>. En el cas d'assignar llistes, cal copiar els valors (sense fer aliasing). Per senzillesa, a la part esquerra d'una assignació només pot aparèixer una variable (no es pot fer, per exemple, l[2] &lt;- 5).</p>
<h2 dir="auto"><a id="user-content-lectura" class="anchor" aria-hidden="true" href="#lectura"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Lectura</h2>
<p dir="auto">La instrucció de lectura ha de llegir un valor enter del canal d'entrada estàndard i enmagatzemar-lo a la variable a la dreta del <code>&lt;?&gt;</code>. Exemple: <code>&lt;?&gt; x</code>.</p>
<h2 dir="auto"><a id="user-content-escriptura" class="anchor" aria-hidden="true" href="#escriptura"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Escriptura</h2>
<p dir="auto">La instrucció d'escriptura ha d'avaluar l'expressió del <code>&lt;!&gt;</code> i escriure-la, en una línia, al canal de sortida estàndard. Exemple: <code>&lt;!&gt; x + y</code>. En el cas d'escriure una llista, cal escriure tots els seus valors entre claudàtors i separats per comes. <code>&lt;!&gt;</code> pot contenir diversos paràmetres, cal escriure cadascun d'ells a la mateix línia, separats per espais. Els paràmetres poden contenir textos (tancats entre cometes dobles). Els textos no apareixen en cap altre lloc.</p>
<h2 dir="auto"><a id="user-content-reproducció" class="anchor" aria-hidden="true" href="#reproducció"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Reproducció</h2>
<p dir="auto">La instrucció de reproducció ha d'avaluar l'expressió del <code>&lt;:&gt;</code>. Si és una nota, l'ha d'afegir a la partitura, amb el valor d'una negra (diferents valors de notes com blanques, rodones o corxeres es deixen per a JSBach2). Si és una llista, ha d'afegir cadascuna de les seves notes (d'esquerra a dreta).</p>
<h2 dir="auto"><a id="user-content-condicional" class="anchor" aria-hidden="true" href="#condicional"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Condicional</h2>
<p dir="auto">La instrucció condicional té la semàntica habitual. El bloc del sinó és optatiu. Exemples: <code>if x == y |: z &lt;- 1 :|</code> i <code>if x == y |: z &lt;- 1 :| else |: z &lt;- 2 :|</code>. Fixeu-vos que els limitadors dels blocs sempre són obligatoris (tant als condicions com als procediments i als <code>while</code>s).</p>
<h2 dir="auto"><a id="user-content-iteració-amb-while" class="anchor" aria-hidden="true" href="#iteració-amb-while"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Iteració amb <code>while</code></h2>
<p dir="auto">La instrucció iterativa amb <code>while</code> té la semàntica habitual. Exemple: <code>while a &gt; 0 |: a &lt;- a / 2 :|</code>.</p>
<h2 dir="auto"><a id="user-content-invocació-de-procediment" class="anchor" aria-hidden="true" href="#invocació-de-procediment"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Invocació de procediment</h2>
<p dir="auto">La crida a un procediment té la semàntica habitual.<br>
Si el nombre de paràmetres passats
no corresponen als declarats, es produeix un error. Els procediments no són
funcions i no poden retornar resultats. Però els procediments es poden cridar
recursivament. La sintàxi és com a Haskell: sense parèntesis ni comes.
Exemple: <code>Escriu x + y 2</code>.</p>
<h2 dir="auto"><a id="user-content-expressions" class="anchor" aria-hidden="true" href="#expressions"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Expressions</h2>
<p dir="auto">Si una variable encara no ha rebut cap valor, el seu valor és zero. Els
operadors aritmètics són els habituals (<code>+</code>, <code>-</code>, <code>*</code>, <code>/</code>, <code>%</code>) i amb la
mateixa prioritat que en C. Evidentment, es poden usar parèntesis. El operadors
relacionals (<code>=</code>, <code>/=</code>, <code>&lt;</code>, <code>&gt;</code>, <code>&lt;=</code>, <code>&gt;=</code>) retornen zero per fals i u per
cert (Boole és posterior a Bach).</p>
<h2 dir="auto"><a id="user-content-llistes" class="anchor" aria-hidden="true" href="#llistes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Llistes</h2>
<p dir="auto">Les llistes s'escriuen entre claudàtors, amb els seus elements separats per comes. Els elements de les llistes només poden ser enters (o notes, que no són altre cosa que constants enteres). Les llistes es passen per referència.
<code>#l</code> retorna la llargada de la llista <code>l</code>.
<code>l[i]</code> consulta l'<code>i</code>-èsim element d'una llista <code>l</code>; cal que <code>i</code> es trobi entre 1 i <code>#l</code>.
<code>l &lt;&lt; x</code> afegeix l'element <code>x</code> al final de la llista <code>l</code>.
<code>8&lt; l[i]</code> elimina l'<code>i</code>-èsim element d'una llista <code>l</code>;
també cal que <code>i</code> es trobi entre 1 i <code>#l</code>.</p>
<h2 dir="auto"><a id="user-content-àmbit-de-visibilitat" class="anchor" aria-hidden="true" href="#àmbit-de-visibilitat"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Àmbit de visibilitat</h2>
<p dir="auto">No importa l'ordre de declaració dels procediments. Les variables són locals a
cada invocació de cada procediment. No hi ha variables globals ni manera
d'accedir a variables d'altres procediments (només a través dels paràmetres).</p>
<h2 dir="auto"><a id="user-content-notes" class="anchor" aria-hidden="true" href="#notes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Notes</h2>
<p dir="auto">JSBach proporciona uns noms que representen les notes blanques d'un piano (els sostinguts i bemolls es deixen per JSBach2). Les tres primeres notes són A0 (La0), B0 (Si0), C1 (Do1). Les tres darreres són A7 (La7), B7 (Si7), C8 (Do8). A <a href="https://ca.wikipedia.org/wiki/Freq%C3%BC%C3%A8ncies_del_piano" rel="nofollow">https://ca.wikipedia.org/wiki/Freq%C3%BC%C3%A8ncies_del_piano</a> teniu una explicació d'aquesta nomemclatura). A més, les notes C, D, E, F, G, A, B
(sense número) son sinónims de C4 (Do central), D4, E4, F4, G4, A4, B4. Les notes de JSBach no són altra cosa que constants, de manera que A0 val 0, B0 val 1, ... i C8 val ???. Així, es pot transposar una nota una octava més amunt o més avall sumant-li o restant-li 7 unitats (sí, els músics són gent peculiar i de 7 notes en diuen una octava).</p>

<p dir="auto">Malgrat que JSBach és força senzill, els programes poden causar molts errors en temps d'execució. <p dir="auto">Si ho voleu, també podeu fer que la música es toqui al final del programa.</p>
<h2 dir="auto"><a id="user-content-extensions" class="anchor" aria-hidden="true" href="#extensions"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Extensions</h2>
<p dir="auto">Podeu extendre el llenguatge amb construccions del vostre gust, a condició de mantenir una compatibilitat estricta amb l'especificació donada (i ser barrocs). A més, cal que documenteu amb precisió les vostres extensions i que creeu programes que les provin i posin de manifest la seva utilitat.</p>
<p dir="auto">Per exemple, podríeu extendre JSBach amb accidentals (notes amb sostinguts o bemolls), operadors lògics, funcions que retornin valors, funcions d'ordre superior...</p>
<p dir="auto">Compte: Les extensions poden portar molta feina, consulteu-les abans amb el vostre professor.</p>
<h2 dir="auto"><a id="user-content-llibreries" class="anchor" aria-hidden="true" href="#llibreries"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Llibreries</h2>
<p dir="auto">Utilitzeu  <code>ANTLR</code> per escriure la gramàtica i l'intèrpret. Podeu utilitzar lliurament qualsevol llibreria estàndard de Python.</p>
<h2 dir="auto"><a id="user-content-programes-externs" class="anchor" aria-hidden="true" href="#programes-externs"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Programes externs</h2>
<p dir="auto">Per generar les partitures, heu d'utilitzar el programa LilyPond.
Lilypond ja genera MIDI i PDF.
Per generar WAV a partir de MIDI heu d'utilitzar <code>timidity++</code>.
Per generar MP3 a partir de WAV heu d'utilitzar <code>ffmpeg</code>.
Tots aquests programes es poden instal·lar facilment en Mac, Linux i (suposo) Windows. El vostre programa es corregirà en un entorn on els binaris <code>lilypond</code>, <code>timidity</code> i <code>ffmpeg</code> es troben al <em>path</em>.</p>
<h2 dir="auto"><a id="user-content-breu-demostració-dels-programes-externs" class="anchor" aria-hidden="true" href="#breu-demostració-dels-programes-externs"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Breu demostració dels programes externs</h2>
<p dir="auto">LilyPond és un programa de tipografia musical.
És programari lliure i forma part del Projecte GNU.
Les partitures es descriuen a partir d'un llenguatge prou complex, però del quan només us cal saber uns rudiments per fer aquesta pràctica.</p>
<p dir="auto">Per exemple, aquest programa <code>exemple.lily</code> en LilyPond descriu una escala
amb buit negres (Do, Re, Mi, Fa, Sol, La, Si, Do) a un tempo de 120:</p>
<div class="snippet-clipboard-content position-relative overflow-auto" data-snippet-clipboard-copy-content="\version &quot;2.22.1&quot;
\score {
    \absolute {
        \tempo 4 = 120
        c'4 d'4 e'4 f'4 g'4 a'4 b'4
    }
    \layout { }
    \midi { }
}"><pre class="notranslate"><code class="notranslate">\version "2.22.1"
\score {
    \absolute {
        \tempo 4 = 120
        c'4 d'4 e'4 f'4 g'4 a'4 b'4
    }
    \layout { }
    \midi { }
}
</code></pre></div>
<p dir="auto">Quan hom executa</p>
<div class="highlight highlight-source-shell position-relative overflow-auto" data-snippet-clipboard-copy-content="lilypond exemple.lily"><pre>lilypond exemple.lily</pre></div>
<p dir="auto">Lilipond genera <code>exemple.pdf</code> i <code>exemple.midi</code> (perquè s'ha donat les comandes <code>\layout</code> i <code>\midi</code> respectivament).</p>
<p dir="auto">TiMidity++ és un altre programa de codi obert que transforma fitxers MIDI en fitxers de so, com ara fitxers WAV. Es pot executar així</p>
<div class="highlight highlight-source-shell position-relative overflow-auto" data-snippet-clipboard-copy-content="timidity -Ow -o exemple.wav exemple.midi"><pre>timidity -Ow -o exemple.wav exemple.midi</pre></div>
<p dir="auto">per tal d'obtenir el fitxer <code>exemple.wav</code>.</p>
<p dir="auto">El programa lliure <code>ffmpeg</code> permet convertir entre diferents formats de vídeo i audio. Es pot executar així</p>
<div class="highlight highlight-source-shell position-relative overflow-auto" data-snippet-clipboard-copy-content="ffmpeg -i exemple.wav -codec:a libmp3lame -qscale:a 2 exemple.mp3"><pre>ffmpeg -i exemple.wav -codec:a libmp3lame -qscale:a 2 exemple.mp3</pre></div>
<p dir="auto">per tal d'obtenir el fitxer <code>exemple.mp3</code>.</p>
<p dir="auto">Finalment, els fitxers MP3 es poden reproduir amb molts reproductors de so. En un Mac, es pot fer així des de la línia de comandes:</p>
<div class="highlight highlight-source-shell position-relative overflow-auto" data-snippet-clipboard-copy-content="afplay exemple.mp3"><pre>afplay exemple.mp3</pre></div>
<h1 dir="auto"><a id="user-content-la-vostra-feina" class="anchor" aria-hidden="true" href="#la-vostra-feina"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>La vostra feina</h1>
<p dir="auto">La vostra feina consisteix en
implementar un (doble) intèrpret de JSBach.</p>
<p dir="auto">Per realitzar la vostra feina heu d'utilitzar Python3 i ANTLR4, tal com s'ha explicat a les classes de laboratori. Per generar les partitures, heu d'utilitzar el programa Lilipond. Per generar els WAV i MP3, els programes Timidity++ i ffmpeg.</p>
<h1 dir="auto"><a id="user-content-consells" class="anchor" aria-hidden="true" href="#consells"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Consells</h1>
<p dir="auto">Per tal de fer la  pràctica, us recomanem de seguir aquests passos:</p>
<ol dir="auto">
<li>
<p dir="auto">Feu tots els exercicis de laboratori inclosos a <a href="https://gebakx.github.io/Python3/compiladors.html#1" rel="nofollow">https://gebakx.github.io/Python3/compiladors.html#1</a>. Us serà molt útil, de debò.</p>
</li>
<li>
<p dir="auto">Escriviu la gramàtica de JSBach restringit a operacions amb enters (sense llistes ni notes ni instrucció de reproducció).</p>
</li>
<li>
<p dir="auto">Escriviu els visitadors per la gramàtica anterior per obtenir una primer versió de l'intèrpret.</p>
</li>
<li>
<p dir="auto">Exteneu la gramàtica amb notes i
instrucció de reproducció.</p>
</li>
<li>
<p dir="auto">Escriviu els visitadors per la gramàtica anterior.
Feu que al acabar generi el fitxer de Lilipond.</p>
</li>
<li>
<p dir="auto">Exteneu la gramàtica, ara amb les llistes i les seves instruccions associades.</p>
</li>
<li>
<p dir="auto">Escriviu els visitadors per la gramàtica anterior.
Amb això ja tindreu tot l'intèrpret.</p>
</li>
<li>
<p dir="auto">Generar els fitxers d'audio al final.</p>
</li>
<li>
<p dir="auto">Escriviu el README.</p>
</li>
<li>
<p dir="auto">Si voleu, realitzeu extensions.</p>
</li>
</ol>
<p dir="auto">Al llarg de tots els passos anteriors, aneu documentant tot allò que no sigui obvi.</p>
<p dir="auto">Deixeu la feina bruta a l'ANTLR: si feu una bona gramàtica, cadascun dels vostres visitadors serà molt curt i molt senzill. Si els vostres visitadors tenen molta lògica, retoqueu la gramàtica, introduint noves regles i/o etiquetes.</p>
<p dir="auto">Feu la pràctica "amb carinyo".</p>
<h1 dir="auto"><a id="user-content-lliurament" class="anchor" aria-hidden="true" href="#lliurament"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Lliurament</h1>
<p dir="auto">Heu de lliurar la vostra pràctica al Racó. Només heu de lliurar un fitxer ZIP
que, al descomprimir-se generi:</p>
<ul dir="auto">
<li>
<p dir="auto">Un fitxer <code>README.md</code> que documenti el vostre projecte.
|:- vegeu, per exemple, <a href="https://www.makeareadme.com/" rel="nofollow">https://www.makeareadme.com/</a>.</p>
</li>
<li>
<p dir="auto">Un fitxer <code>jsbach.g4</code> amb la gramàtica del LP.</p>
</li>
<li>
<p dir="auto">Un fitxer <code>jsbach.py</code> amb el programa de l'intèrpret, incloent els seus visitadors.</p>
</li>
<li>
<p dir="auto">Si heu fet extensions, podeu afegir fitxers <code>test-*.jsb</code> com a exemples i jocs de proves.</p>
</li>
<li>
<p dir="auto">Res més. De debò, res més. Tampoc directoris. Ni subdirectoris. Ni <code>.git</code>s, ni <code>.MACOS_XXX</code>... Res més, collons!</p>
</li>
</ul>
<p dir="auto">Els vostres fitxers de codi en Python han de seguir les regles d’estı́l PEP8, tot i que podeu oblidar les restriccions sobre la llargada màxima de les lı́nies. Podeu utilitzar els paquets <code>pep8</code> o <code>autopep8</code> o <a href="http://pep8online.com/" rel="nofollow">http://pep8online.com/</a> per assegurar-vos que seguiu aquestes regles d’estı́l. L’ús de tabuladors en el codi queda prohibit (zero directe). Els vostres programes en JSBach han de seguir l'estil exposat en aquest document.</p>
<p dir="auto">El termini de lliurament és el <strong>dimecres 8 de juny a les 13:00</strong>.</p>

no pengeu el vostre projecte en repositoris públics.</p>
